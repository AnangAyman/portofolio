NusaCrop Cloud Infrastructure

A scalable, highly available Google Cloud architecture designed to power AI driven agricultural disease detection across Indonesia.

Role

Cloud Architect Engineer

Timeline

Team Capstone Project

Domain

GCP, Scalability, DevOps

Core Tech

Docker, Cloud Run, Cloud SQL

The Challenge

NusaCrop is tackling a massive agricultural crisis in Indonesia, aiming to empower millions of distributed local farmers with instant plant disease detection via mobile. However, delivering real-time, compute-heavy image recognition inferences across a wildly fluctuating user base in remote regions requires an ironclad backend architecture.

The primary engineering hurdle was avoiding a monolith structure that would crash under peak harvest-season load, all while minimizing latency during tensor model execution originating from unreliable mobile networks.

The Solution

I engineered a fully decoupled microservices architecture deployed entirely on Google Cloud Platform (GCP). By splitting the application into an API Gateway, an Authentication Service, and an isolated Machine Learning Inference worker, the system guarantees that high-CPU image processing does not throttle core application routing.

The entire stack is containerized using Docker, managed through a strictly automated CI/CD pipeline on GitHub Actions, and deployed onto Cloud Run for seamless, zero-maintenance auto-scaling. Data persistence is reliably handled through horizontally scaling Cloud SQL instances.

Cloud Run & Auto-Scaling

Leveraged fully managed GCP compute services allowing backend containers to automatically scale out from zero to hundreds of concurrent instances during peak traffic.

Containerization

Engineered highly optimized Docker images stripping away unnecessary dependencies to ensure the ML Inference microservice boots and responses with sub-second latency.

CI/CD Pipeline

Implemented rigorous GitHub Action workflows that automatically build, test, and push code changes directly to GCP Artifact Registry on every main branch merge.

The Outcome

The architecture effectively processed user uploads and generated reliable predictions within a 99.9% uptime SLA. The decoupling ensured that developers could continuously update the CNN models without ever risking downtime to the primary user authentication and data-fetching APIs.
